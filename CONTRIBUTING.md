# Contributing to Sharat Raj's Personal Website

Thank you for your interest in contributing to this personal website! While this is primarily a personal project, I welcome contributions that improve the site's functionality, fix bugs, or enhance the user experience.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. **Check existing issues** to see if it's already been reported
2. **Create a new issue** with a clear title and description
3. **Include details** such as:
   - What you expected to happen
   - What actually happened
   - Steps to reproduce the issue
   - Browser and operating system information

### Suggesting Enhancements

To suggest new features or improvements:

1. **Open an issue** with the "enhancement" label
2. **Describe the feature** and why it would be useful
3. **Consider the scope** - keep suggestions relevant to a personal website/blog

### Pull Requests

I appreciate pull requests that:

- Fix bugs
- Improve documentation
- Enhance accessibility
- Optimize performance
- Fix typos or formatting issues

####Steps to Submit a Pull Request:

1. **Fork the repository**
   ```bash
   git clone https://github.com/s-raj/s-raj.github.io.git
   cd s-raj.github.io
   ```

2. **Create a new branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow the existing code style
   - Test your changes locally
   - Ensure the site builds without errors

4. **Commit your changes**
   ```bash
   git add .
   git commit -m "Description of your changes"
   ```

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a pull request**
   - Provide a clear description of the changes
   - Reference any related issues
   - Include screenshots if applicable

## Development Guidelines

### Code Style

- Follow the existing code style and conventions
- Use meaningful variable and function names
- Comment complex logic
- Keep files organized according to the directory structure

### Testing

Before submitting a pull request:

1. **Test locally**
   ```bash
   bundle exec jekyll serve
   ```

2. **Check the site** at `http://localhost:4000`
3. **Test on multiple browsers** if possible
4. **Verify mobile responsiveness**

### Content Guidelines

**Note:** Please do not modify personal content (blog posts, about me page, personal information) unless fixing factual errors or typos that I should be aware of.

You can contribute to:
- Documentation files
- Configuration improvements
- Template enhancements
- CSS/JavaScript improvements
- Accessibility features

## What I'm Looking For

Helpful contributions include:

- **Bug fixes** - Broken links, display issues, errors
- **Performance improvements** - Faster load times, optimization
- **Accessibility enhancements** - Better screen reader support, keyboard navigation
- **Documentation** - Clearer instructions, better examples
- **Theme improvements** - Better layouts, responsive design tweaks

## What to Avoid

Please don't:

- Modify personal blog content without permission
- Change personal information or contact details
- Add tracking or analytics beyond what's configured
- Include external dependencies without discussion
- Make large-scale changes without prior discussion
